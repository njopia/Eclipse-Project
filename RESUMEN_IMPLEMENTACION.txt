================================================================================
                               FUNCIONALIDAD
================================================================================

HUD2 - TEAM BONUSES (Actualizado cada 0.5 segundos)
├─ Team Speed Boost: Muestra tiempo restante en MM:SS
│  ├─ Ej: "Speed Boost: 5:30"
│  └─ Obtiene dato de: GetTeamSpeedBoostRemaining(client)
├─ Team Heal: Muestra cooldown en segundos
│  ├─ Ej: "Team Heal CD: 45s" o "Team Heal: READY"
│  └─ Obtiene dato de: GetTeamHealCooldown(client)
└─ Survivors: Contador de supervivientes activos
   ├─ Ej: "Survivors: 4/4"
   └─ Calcula dinámicamente durante cada actualización

HUD3 - DEPLOYABLES (Actualizado cada 0.5 segundos)
├─ UV Light: Muestra tiempo restante en segundos
│  ├─ Ej: "UV Light: 250s"
│  └─ Obtiene dato de: IsUVLightActive(client) + GetUVLightRemaining(client)
├─ Healing Station: Muestra tiempo restante en segundos
│  ├─ Ej: "Healing Station: 180s"
│  └─ Obtiene dato de: IsHealingStationActive(client) + GetHealingStationRemaining(client)
└─ Estado de inactividad
   └─ "None active" si no hay deployables

================================================================================
                          EXTENSIONES FUTURAS
================================================================================

El sistema es totalmente extensible. Para agregar más datos:

1. En el archivo feature correspondiente (ej: uv-light.feature.sp):
   ├─ Agregar variable para tiempo final: static float g_fUVLightEnd[MAXPLAYERS + 1];
   ├─ Agregar funciones stock:
   │  ├─ stock bool IsUVLightActive(int client)
   │  └─ stock float GetUVLightRemaining(int client)
   └─ Publicar en el .inc (include file)

2. En hud-system-display.feature.sp:
   ├─ Agregar forward declaration
   ├─ Utilizar en BuildDeployablesHUDText()
   └─ El resto funciona automáticamente

3. Las funciones se detectan automáticamente en tiempo de compilación
